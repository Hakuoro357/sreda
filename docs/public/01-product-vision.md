# Product Vision

## Цель

Построить платформу персональных ассистентов, которая:
- общается с пользователями через мессенджеры;
- умеет безопасно хранить персональные данные;
- строго разделяет данные, память и рабочие контуры разных клиентов;
- может работать как облачный сервис и как персональная коробка в контуре клиента.

## Основные сценарии

1. Пользователь пишет ассистенту в `Telegram`.
2. Ассистент помнит контекст, профиль, предпочтения и ограничения.
3. Ассистент умеет запускать рабочие сценарии:
- отвечать на вопросы;
- собирать информацию;
- обрабатывать события;
- запускать фоновые процессы по расписанию;
- присылать сводки и уведомления.

## Бизнес-требования

- персональный ассистент для каждого пользователя;
- в будущем поддержка команды, workspace и общего контекста;
- SaaS-модель для обычных клиентов;
- отдельный single-tenant контур для чувствительных клиентов;
- понятная эксплуатация и поддержка.

## Нефункциональные требования

- строгая изоляция по `tenant` и `workspace`;
- шифрование чувствительных данных;
- аудит действий системы;
- управляемые фоновые процессы;
- возможность human approval для рискованных действий;
- переносимость в on-prem.

## Первый запуск

На первом запуске стоит ограничить scope:
- только `Telegram`;
- только текст;
- один основной тип ассистента;
- один runtime;
- одна БД;
- без сложного multi-agent оркестратора;
- без сложной векторной памяти;
- без полноценной MAX-интеграции.
