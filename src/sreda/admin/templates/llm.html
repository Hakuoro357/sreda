LLM: primary и fallback

Переключение провайдера живое — применяется со следующего хода любого активного диалога, рестарт сервисов не нужен. Fallback активируется автоматически на 429/5xx/timeout primary-провайдера и прозрачно переигрывает тот же запрос на запасном.

Доступные провайдеры

Приоритет значений
  1. Значения из этой формы (сохраняются в таблицу runtime_config).
  2. Env-переменные SREDA_CHAT_PROVIDER / SREDA_CHAT_FALLBACK_PROVIDER.
  3. Дефолт mimo для primary, без fallback.

Кэш значений — 5 секунд в памяти процесса. Изменения видны в течение ~5с в каждом воркере.
