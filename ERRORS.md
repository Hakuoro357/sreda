# ERRORS

## Назначение

Типовые ошибки в работе по `Среде` и способы обхода.

## Ошибка: смешение `Среды` и `Ассистента`

Проблема:
- в docs, путях и терминологии смешивались два разных проекта.

Как обходить:
- в этом workspace держать только `Среду`;
- не использовать термины и пути другого проекта;
- проверять docs и runtime paths на смешение.

## Ошибка: публикация internal docs в публичный repo

Проблема:
- `docs/internal` однажды был запушен в публичный `sreda`.

Как обходить:
- в публичный репозиторий класть только `sreda/docs/public`;
- перед push отдельно проверять, что `docs/internal` не staged;
- при работе с docs помнить структуру:
  - `docs/public`
  - `docs/internal`

## Ошибка: реальные `chat_id` в docs, коде и тестах

Проблема:
- реальные Telegram `chat_id` попали в README, seed scripts и tests.

Как обходить:
- в docs использовать placeholders;
- в seed/test коде использовать только тестовые/демо значения;
- перед commit отдельно искать реальные `chat_id`.

## Ошибка: PowerShell ломает сложные команды

Проблема:
- one-liner со спецсимволами и списками pathspec часто ломаются в `PowerShell`.

Как обходить:
- использовать временный скрипт;
- избегать сложных inline-команд;
- для git-команд с несколькими pathspec при проблемах либо вызывать их проще, либо переходить на безопасный скрипт.

## Ошибка: inline Python через ssh one-liner

Проблема:
- конструкция вида `ssh "... python - <<'PY' ..."` в этой среде снова ломала кавычки и искажала SQL/Python-код.

Как обходить:
- не запускать многострочный Python через `ssh` inline;
- писать отдельный временный `.py` файл;
- заливать его на хост и запускать как обычный файл.

## Ошибка: push запускался раньше завершения commit

Проблема:
- `push` стартовал до того, как commit реально завершился.

Как обходить:
- commit и push делать строго отдельными последовательными шагами;
- после commit проверять `log` или `branch -vv`, потом уже push.

## Ошибка: PowerShell не принимает `&&`

Проблема:
- в этой среде `PowerShell` не поддерживает `&&` как разделитель команд и валится на склеенных многошаговых командах.

Как обходить:
- не использовать `&&` в командах для этой среды;
- выполнять шаги последовательно отдельными вызовами;
- если шагов много, выносить их во временный скрипт.
